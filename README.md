# 🎬 Subtitle Sidekick — כתוביות עברית להרצאות

<p align="center">
  <img src="https://img.shields.io/badge/Python-3.12-blue" alt="Python 3.12" />
  <img src="https://img.shields.io/badge/Whisper-ivrit--ai-c56a43" alt="ivrit-ai Whisper" />

  <img src="https://img.shields.io/badge/Windows-desktop-c59a3f" alt="Windows desktop" />
</p>

*A local, free, privacy-first desktop app that transcribes recorded Hebrew lectures into perfectly-synced subtitles — no cloud, no account, no cost.*

אפליקציה שמתמללת הרצאות (Moodle / כל וידאו) לעברית ומציגה כתוביות **מסונכרנות בול לאודיו**. רץ מקומית על המחשב שלך — בלי ענן, בלי חשבונות, בלי עלות.

> מבוסס Whisper עם מודל עברי ייעודי (ivrit-ai). אומת: מתמלל עברית במדויק, כולל פיסוק.

## ⭐ תכונות עיקריות

- 🏠 **מסך בית** — כפתור "תמלול חדש" בולט, "המשך מאיפה שעצרת", שורת סטטיסטיקה וקורסים אחרונים.
- 🌍 **בחירת שפה** — עברית (מודל ייעודי), אנגלית, או זיהוי אוטומטי — לכל הרצאה בנפרד.
- 📚 **ספריית קורסים** — כל הרצאה מאורגנת לפי קורס, עם תפריט פעולות לכל הרצאה.
- ⬇️ **הורדה מקישור** (YouTube / Moodle) — נכנס אוטומטית לתור התמלול.
- 🗂️ **תור תמלול** — אפשר לגרור כמה קבצים יחד, מתמללים אחד-אחד ברקע.
- ⚡ **זיהוי GPU אוטומטי** — NVIDIA מזורז אוטומטית; בלי GPU, מתג "מהיר" למודל קליל יותר.
- 🔎 **חיפוש בכתוביות** + קפיצה ישירה לרגע בהרצאה.
- ✏️ **עורך כתוביות** מובנה לתיקון טעויות תמלול.
- 📄 **ייצוא** לתמליל TXT / Word.
- ⛶ **נגן מלא**: מסך מלא, דילוג 10 שניות, מהירות ניגון 0.5x–2x, ושליטה בגודל/רקע הכתוביות (🅰).
- 📖 **מדריך שימוש מובנה** — כפתור ❓ בסרגל העליון פותח מדריך בתוך האפליקציה.
- 🌐 **נגן עצמאי בדפדפן** — נוצר לכל הרצאה, עובד בלי האפליקציה.
- 🔒 **מקומי כברירת מחדל** — בלי ענן, בלי חשבון, האודיו לא עוזב את המחשב (אלא אם בוחרים מצב ענן).

## 📥 התקנה (פעם אחת)

1. **התקן Python 3.12** — [הורדה מכאן](https://www.python.org/downloads/release/python-3120/). בהתקנה **סמן ✅ "Add Python to PATH"**.
2. **הורד את הפרויקט** — כפתור ירוק **Code** → **Download ZIP** → חלץ לתיקייה.
3. **לחיצה כפולה על `run.bat`** — זה מתקין הכל לבד (כמה דקות, פעם אחת) ויוצר אייקון **"Subtitle Sidekick"** בשולחן העבודה.

זהו. מהפעם הבאה — פשוט לחיצה כפולה על האייקון בשולחן העבודה. **בלי שורת פקודה.**

> בהרצה הראשונה יורד המודל העברי (~1.5GB) אוטומטית. אם יש כרטיס מסך NVIDIA — האפליקציה תזהה ותשתמש בו אוטומטית (מהיר בהרבה).

## ⭐ איך משתמשים

1. **הורד הרצאה** — או הדביקו קישור ישיר (YouTube / Moodle / כל קישור-וידאו ישיר) ולחצו **הורד**, או הורידו בעצמכם (למשל בעזרת תוסף הכרום Video DownloadHelper).
2. פתח את האפליקציה (האייקון בשולחן העבודה, או סימנייה ל-`subsidekick://open` בכרום) — תיפתח **למסך הבית**.
3. מהבית: **תמלול חדש** → **גררו או בחרו** קובץ (אפשר כמה יחד — נכנסים ל**תור** שמתמלל סדרתית). לפני ההעלאה אפשר לבחור **קורס** ו**שפת הרצאה** (עברית / אנגלית / זיהוי אוטומטי) → **צור כתוביות** → **▶ צפה עם כתוביות**.

כל הרצאה שתומללה נרשמת אוטומטית לתפריט הצד (☰) — מאורגנת לפי קורסים, עם תפריט פעולות (⋯) לכל הרצאה: שינוי שם, העברה לקורס אחר, פתיחה בדפדפן, הסרה.

> למחשב איטי בלי GPU: מתג **"מהיר"** מתמלל פי כמה מהר (מודל קל יותר). הרצאה ארוכה על מעבד לוקחת זמן — אפשר להשאיר רץ ברקע (שבשבת ההתקדמות גלויה מכל מסך).

## ☁ מצב "שרת מהיר" (אופציונלי)

<details>
<summary>תמלול דרך שרת GPU שלכם — מהיר מאוד, אבל לא חובה (לחצו להרחבה)</summary>

ברירת המחדל היא ותישאר **מקומית** — בלי קונפיג, בלי חשבון. מי שרוצה תמלול מהיר במיוחד יכול לבחור **"☁ שרת GPU (ענן)"** במסך ההעלאה, אחרי הקמת endpoint אישי ב-RunPod (חד-פעמי, כ-5 דקות — בלי Docker).

חשוב: **זה לא שרת משותף**. לכל משתמש endpoint ו-API key אישיים משלו ב-RunPod, והאודיו עולה רק לשרת שלכם — אין שום כתובת או מפתח מוטבעים באפליקציה.

המסלול המומלץ משתמש ב-image מוכן שכבר בנוי ופומבי, כך שאין צורך להתקין Docker או לבנות כלום. (למי שרוצה שליטה מלאה — יש בסוף נספח לבנייה עצמית.)

ה-image כולל **שני מודלים**: מודל עברית ייעודי, ומודל כללי לאנגלית/זיהוי-אוטומטי — כך שתמלול ענן עובד בכל השפות בהן תומך Whisper, לא רק עברית.

### מה צריך מראש

- ☑ חשבון [RunPod](https://www.runpod.io/) עם קרדיט (אפילו $5-10 מספיקים להתחלה).

### שלב 1 — מפתח API ב-RunPod

1. נכנסים ל-[runpod.io](https://www.runpod.io/) ומתחברים.
2. תפריט צד → **Settings** → לשונית **API Keys** → **+ Create API Key**.
3. נותנים שם (לדוגמה `subtitle-sidekick`) → **Create**.
4. **מעתיקים את המפתח עכשיו** — הוא מוצג פעם אחת בלבד.

### שלב 2 — יצירת ה-Endpoint עם ה-image המוכן

1. נכנסים ב-RunPod לתפריט הצד → **Serverless** → **+ New Endpoint**.
2. בוחרים **Custom Source** (Docker image).
3. בשדה ה-image מדביקים בדיוק: `itaylot/subtitle-sidekick-server:latest`
4. בוחרים GPU — אופציה זולה שמספיקה (RTX A4000 או A5000); אין צורך ב-GPU חזק לתמלול.
5. קובעים Workers: ‏Min Workers = `0` (לא משלמים כשלא משתמשים), ו-Max Workers = `4` (מאפשר תמלול מקבילי של מקטעי האודיו — מהיר יותר; אפשר גם `1` לשימוש חסכוני).
6. לוחצים **Create Endpoint**.
7. בדף ה-endpoint שנפתח, בראש העמוד מופיע **Endpoint ID** (מחרוזת אותיות/ספרות) — מעתיקים אותו.

### שלב 3 — הזנת הפרטים באפליקציה

ב-Subtitle Sidekick: פותחים את תפריט הצד (☰) → **⚙ הגדרות** → אזור "שרת ענן":
- **כתובת השרת:** `https://api.runpod.ai/v2/ENDPOINT_ID` (מחליפים `ENDPOINT_ID` במה שהעתקתם בשלב 2.7)
- **מפתח API:** המפתח משלב 1.4
- **מחיר ה-GPU לשעה:** המחיר שמופיע ב-RunPod עבור ה-GPU שבחרתם (לחישוב עלות מוערכת).
- **שמירת הגדרות השרת**.

זהו — מהעלאה הבאה, בוחרים **☁ שרת GPU (ענן)** מתפריט מצב התמלול. ה-API key נשמר במחשב ולא צריך להזין אותו מחדש.

**עלות:** RunPod Serverless מחייב רק על זמן עיבוד בפועל (scale-to-zero — אין חיוב כשהשרת לא בשימוש). האפליקציה מציגה **עלות מוערכת** מצטברת לפי המחיר לשעה שהזנתם; זו הערכה בלבד — RunPod מחייב גם על זמני התנעה (cold start) ועיגול, כך שייתכן הפרש קל מהחשבון בפועל.

אם השרת לא מוגדר, לא זמין, או מחזיר שגיאה — תוצג הודעה ברורה, ואפשר לבחור מצב מקומי ולהמשיך בלי לאבד את הקובץ.

### נספח למתקדמים — בניית image משלכם

<details>
<summary>למי שרוצה שליטה מלאה על קוד השרת או image פרטי (לחצו להרחבה)</summary>

דורש [Docker Desktop](https://www.docker.com/products/docker-desktop/) מותקן ופתוח, וחשבון חינמי ב-[Docker Hub](https://hub.docker.com/).

> ⚠️ **אם תיקיית הפרויקט נמצאת תחת OneDrive**: Docker עלול להיכשל בקריאת קבצים שסונכרנו כ"placeholder בענן" עם שגיאה כמו `invalid file request Dockerfile`. אם זה קורה — העתיקו את תיקיית `runpod_server` למקום מחוץ ל-OneDrive (למשל `C:\runpod_server`) ובנו משם.

בטרמינל, מתוך תיקיית `runpod_server`:

```bash
docker login
docker build -t YOUR_DOCKERHUB_USERNAME/subtitle-sidekick-server:v1 .
docker push YOUR_DOCKERHUB_USERNAME/subtitle-sidekick-server:v1
```

מחליפים את `YOUR_DOCKERHUB_USERNAME` בשם המשתמש שלכם ב-Docker Hub. שלב ה-`build` מוריד את המודל העברי (כמה GB) ואופה אותו ל-image — **לוקח 20-30 דקות**, חד-פעמי. אחר כך משתמשים בשם המלא (`YOUR_DOCKERHUB_USERNAME/subtitle-sidekick-server:v1`) בשלב 2.3 למעלה במקום ה-image המוכן.

> 💡 תיוג גרסה (`:v1`, `:v2`) במקום `:latest` נותן שליטה: עדכון image עתידי לא יחול על endpoints קיימים עד שתשנו אותם לגרסה החדשה.

</details>

## 🛠️ אתגרים טכניים שנפתרו

<details>
<summary>פירוט עבור מי שמתעניין בצד ההנדסי (לחצו להרחבה)</summary>

- **באג רקורסיה אינסופית ב-pywebview:** חשיפת אובייקט החלון (`Window`) על מחלקת ה-API שמשמשת כגשר ל-JS גרמה ל-pywebview לסרוק אותו רקורסיבית בעת בניית הגשר — קריסה/תקיעה אקראית, בעיקר בהזזת החלון. אותרה ע"י קריאת מקור הספרייה, ותוקנה ע"י קונבנציית `_` בשם התכונה (ש-pywebview מדלג עליה אוטומטית).
- **חסימת ניגון וידאו מ-`file://`:** pywebview מגיש את ה-UI דרך שרת HTTP מקומי, והדפדפן חוסם טעינת מדיה מ-`file://` כשהמקור הוא `http` (מסך שחור, 0:00 קפוא). הפתרון: שרת מדיה מקומי פנימי ב-Python שתומך ב-HTTP Range (כדי לאפשר קפיצה בווידאו), עם URL ייעודי שמוזרם ל-JS.
- **כתוביות שנעלמות במסך מלא:** כשעברנו לנגן הדפדפן הנייטיב, ה-overlay הידני של הכתוביות לא נכנס ל-fullscreen (שמראה רק את אלמנט ה-`<video>`). נפתר במעבר ל-WebVTT `<track>` נייטיב — הדפדפן מצייר את הכתוביות בעצמו ושומר אותן גלויות בכל מצב.
- **מדידת ביצועים על חומרה אמיתית (CPU בלבד, בלי GPU נתמך):** השוואה בין מודל "מדויק" ל-"מהיר" על קליפ של 7.7 דקות — הראתה פער של פי 7-8+ בזמן ריצה, מה שהוביל להחלטה לתת ברירת מחדל מודעת-חומרה במקום הנחה גנרית.

</details>

## למתקדמים — שורת פקודה

<details>
<summary>תמלול ישיר ל-SRT (לשילוב ב-VLC או בכל נגן)</summary>

```bash
py -3.12 -m venv .venv && .venv\Scripts\activate    # macOS/Linux: python3.12 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python tools/transcribe_to_srt.py "הרצאה.mp4"        # יוצר הרצאה.srt
```
לצפייה: פותחים את הווידאו ב-VLC (אם ה-`.srt` ליד הווידאו באותו שם — נטען אוטומטית), או ב-[`tools/player.html`](tools/player.html).
</details>

## 📁 מבנה

```
app.py                      משגר חלון pywebview + bridge ל-Python
engine.py                   מנוע התמלול (faster-whisper / ivrit-ai) + ייצוא
worker.py                   מריץ תמלול מקומי בתהליך נפרד (כדי שניתן לעצור/לבטל)
cloud_backend.py            תמלול דרך שרת GPU אישי (RunPod) — ראו מצב "שרת מהיר"
ui/                         ממשק המשתמש (HTML/CSS/JS + פונטים מקומיים)
run.bat                     משגר (התקנה אוטומטית + אייקון בפעם הראשונה)
requirements.txt            תלויות
tools/
  transcribe_to_srt.py      תמלול משורת פקודה: קובץ → SRT
  player.html               נגן מקומי לצפייה עם כתוביות
  make_shortcut.py          יוצר את האייקון בשולחן העבודה
  register_protocol.py      רושם subsidekick:// כדי שסימנייה בכרום תפתח את האפליקציה
runpod_server/               קוד השרת (Docker image) למצב "שרת מהיר" האופציונלי
  handler.py                פונקציית התמלול שרצה על השרת (עברית + מודל כללי לאנגלית)
  Dockerfile                בניית ה-image (אופה את מודלי התמלול בתוכו)
```
